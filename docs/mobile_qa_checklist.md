# Mobile QA Checklist

Smart Task Flow 모바일 화면을 빠르게 회귀 점검하기 위한 최소 체크리스트입니다.

## 실행 전

- `npm run smoke:mobile`로 렌더러 스모크가 통과하는지 확인합니다.
- 실제 브라우저 확인은 390px 전후 모바일 폭과 실제 데이터가 있는 트래커에서 진행합니다.

## 핵심 화면

### 목록

- 업무 카드가 가로 스크롤 없이 표시됩니다.
- 긴 업무명은 두 줄 안에서 자연스럽게 줄임 처리됩니다.
- 하위 업무 토글은 `보기`/`접기` 상태와 실제 펼침 상태가 일치합니다.
- 지연 또는 높은 우선순위 업무가 시각적으로 구분됩니다.

### 캘린더 월간

- 선택 월의 날짜별 업무 카드가 표시됩니다.
- 오늘 표시와 날짜 헤더가 카드와 겹치지 않습니다.
- 하위 업무 시작 카드가 본 업무 카드 안에서 과도하게 길어지지 않습니다.

### 캘린더 년간

- 월 축과 간트 막대가 가로로 잘리지 않습니다.
- 업무가 많은 경우 `주요 n/전체` 또는 `총 n개 · 본 업무` 문구가 자연스럽게 표시됩니다.
- 긴 업무명은 막대 안에서 줄임 처리되고 터치 대상이 너무 작아지지 않습니다.

### 월별 요약

- 모바일 배경은 밝은 테마에서 흰색 계열로 유지됩니다.
- 진행 메모 섹션이 요약 상단에 먼저 보입니다.
- `전체`/`본 업무`/`하위 업무` 필터와 작성자 필터가 한 줄 흐름을 해치지 않습니다.

## 자동화 범위

`npm run smoke:mobile`은 `jsdom`으로 다음을 확인합니다.

- 모바일 목록 카드 렌더링
- 모바일 캘린더 월간 카드 렌더링
- 모바일 캘린더 년간 간트와 밀집 상태 배지 렌더링
- 모바일 월별 요약 메모 및 배경 클래스 렌더링

실제 터치감, 시각적 겹침, 폰트 렌더링은 브라우저에서 최종 확인합니다.
